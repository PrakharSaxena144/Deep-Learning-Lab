P1 · Week 1
Perceptron — Binary Classification from Scratch
"The simplest possible learner — a single neuron with a threshold."

WHEN Use a Perceptron

Use it when your two classes can be separated by a straight line (linearly separable data). It will literally not work — never converge — if the data is not separable (like XOR). Classic use cases: AND gate, OR gate, simple 2D classification.

Analogy: Imagine drawing one straight line between red and blue dots. That's all a perceptron can do.

HOW The Bias Trick (X_aug)

Instead of writing w·x + b separately, you stick a column of 1s at the front of X and make the bias just another weight w[0]. This simplifies the update rule — you only update one vector w.

X_aug = np.hstack((np.ones((X.shape[0], 1)), X))
# X was shape (n, 2) → X_aug is (n, 3)
# w[0] is the bias, w[1], w[2] are the real weights

HOW The Perceptron Update Rule

The rule is: if a point is misclassified, push the weight vector toward it (or away from it).

# y[i] * dot(w, x) <= 0  means misclassified
if y[i] * np.dot(self.w, X[i]) <= 0:
    self.w += y[i] * X[i]
# If y=+1: w += x  (move w closer to x)
# If y=-1: w -= x  (move w away from x)
Why does this work? The decision boundary is w·x = 0 (a line through origin). If a +1 point is on the wrong side, adding its vector to w rotates the boundary toward it. It's geometry, not calculus.

HOW Decision Boundary Plot

The boundary satisfies w0 + w1·x1 + w2·x2 = 0. Solve for x2: x2 = -(w0 + w1·x1) / w2. Generate x1 values → calculate x2 → plot the line.

HOW XOR with Perceptron Network (hardcoded)

XOR is not linearly separable so one perceptron can't do it. You need 2 layers. The trick: XOR(A,B) = AND(OR(A,B), NAND(A,B)). You hardcode the weights based on the logic.

Input (A, B) → OR gate: weights=[1,1], bias=-0.5 → output if A or B is 1 → NAND gate: weights=[-1,-1], bias=1.5 → output if NOT both are 1 → AND gate: weights=[1,1], bias=-1.5 → combine both outputs

You use the step function (output 1 if z≥0, else 0) at each gate.

P2 · Week 2
MLP for Boolean Functions — Step vs Sigmoid
"Can a single output neuron learn all 16 Boolean functions when given the right hidden representation?"

WHEN Use This Pattern

When you want to train a single output perceptron on top of a fixed hidden layer. The hidden layer here is np.eye(4) — the identity matrix — which means each input is already perfectly separated (one-hot). The output neuron just learns the right combination.

WHY HIDDEN_OUTPUTS = np.eye(4)

This is the clever part. Boolean functions take 2 inputs → 4 combinations: (0,0), (0,1), (1,0), (1,1). Instead of using these directly, the "hidden layer" outputs a 4D one-hot vector for each input — [1,0,0,0], [0,1,0,0], etc. This means the output perceptron receives 4 perfectly distinct patterns and only needs to learn which ones to fire for.

Analogy: Instead of teaching someone "if cold AND raining, wear jacket", you give them 4 labeled boxes. They just pick which boxes say YES for the function they're learning.

HOW Two Variants — Step vs Sigmoid

FeatureStep (Threshold)Sigmoid
OutputHard 0 or 1Soft 0.0–1.0
Update conditionOnly when wrong (yh ≠ target)Always update (error is always non-zero)
Update rulew += (target - yh) * xSame formula, but yh is now a probability
ConvergenceStops cleanly when all correctContinuously refines toward target
# Step: only update if misclassified
if yh != target:
    p.w += (target - yh) * x

# Sigmoid: always update (gradient is never exactly 0)
yh = sigmoid(dot(w,x) + b)   # yh is between 0 and 1
p.w += (target - yh) * x     # update every single step

NOTE This is NOT backprop

The update rule here looks like gradient descent but it's actually the Perceptron Learning Algorithm (PLA) — there's no chain rule, no hidden layer gradients. The hidden layer is frozen. Only the output neuron's weights change.

P3 · Week 3
Gradient Descent — Visualizing the Loss Surface
"Before training any neural network, understand what GD is actually doing to a surface."

WHEN Use This

When you need to demonstrate that gradient descent finds a minimum of a function by following the slope downhill. This is the conceptual foundation for all of backprop. Also use this pattern whenever you're asked to minimize a given formula manually.

HOW The Function and Its Gradients

The function is f(x,y) = x² + y² + 10sin(x) + 10cos(y). To run gradient descent, you need the partial derivatives — computed by hand:

# ∂f/∂x: derivative of x² is 2x, derivative of 10sin(x) is 10cos(x)
dfdx = 2*x + 10*np.cos(x)

# ∂f/∂y: derivative of y² is 2y, derivative of 10cos(y) is -10sin(y)
dfdy = 2*y - 10*np.sin(y)

# Update: move opposite to gradient
x -= lr * dfdx
y -= lr * dfdy
Analogy: You're standing on a hilly terrain blindfolded. You can only feel the slope under your feet. You always step in the steepest downhill direction. That's gradient descent.

HOW The Path Array

The function returns a numpy array of shape (iters+1, 3) — each row is (x, y, f(x,y)). Then you can plot path[:,0] vs path[:,1] on the contour to see the trajectory, and path[:,2] to see loss decreasing.

WHY Compare Learning Rates

Learning RateBehaviourSign
Too small (0.001)Takes forever, barely movesBad — slow convergence
Just right (0.05)Smooth descent to minimumGood
Too large (0.2+)Oscillates, may divergeBad — overshooting

Different starting points (initial values) can lead to different local minima because this function is non-convex (it has multiple valleys due to sin/cos).

HOW Contour vs 3D Plot

# Contour (top-down view) — best for trajectory
plt.contour(X, Y, Z, levels=40)   # X,Y,Z are meshgrid arrays
plt.plot(path[:,0], path[:,1])   # overlay the (x,y) path

# 3D surface — best for showing the "hills and valleys"
ax = fig.add_subplot(111, projection='3d')
ax.plot_surface(X, Y, Z, alpha=0.5)
ax.plot(xs, ys, zs)  # 3D path = (x, y, f(x,y))
P4 · Week 4
1-Hidden-Layer NN — 4 Activation Functions Compared
"Same architecture, same data, same training — only the hidden activation changes. Which one wins?"

WHEN Use This Architecture

Use a single hidden layer NN for any binary classification task. This is the template for P4, P7, and most OTS questions. You pass input through a hidden layer (with the activation you choose), then a sigmoid output layer for probability.

Input X (m×2) → Z1 = X @ W1 + b1 (m×48) — linear combination → A1 = activate(Z1) (m×48) — non-linearity applied → Z2 = A1 @ W2 + b2 (m×1) — output pre-activation → A2 = sigmoid(Z2) (m×1) — probability [0,1]

WHY Each Activation and Its Derivative

ActivationFormulaDerivativeProblem
Sigmoid1/(1+e⁻ˣ)s·(1−s)Vanishing gradient for large |x|
Tanhtanh(x)1 − tanh(x)²Also vanishes, but zero-centered (better)
ReLUmax(0,x)1 if x>0 else 0Dying ReLU (dead neurons if always negative)
Leaky ReLUx if x>0 else 0.01x1 if x>0 else 0.01Fixes dying ReLU

The derivative is used in backprop: dZ1 = dA1 * act_deriv(Z1). The chain rule says multiply the upstream gradient by how much the activation was "open" at that point.

HOW Binary Cross-Entropy Loss

Used when output is a probability (sigmoid). Penalises confident wrong predictions very heavily.

def bce(y, yh):
    eps = 1e-9   # prevent log(0)
    return -np.mean(y*np.log(yh+eps) + (1-y)*np.log(1-yh+eps))
# If y=1 and yh≈1: log(1)≈0 → small loss ✓
# If y=1 and yh≈0: log(0)→-∞ → huge loss ✓

HOW Mini-batch Training

Instead of computing gradients on all 1500 samples every step (full batch) or 1 sample (SGD), mini-batch uses a random subset of 64 each time. This is faster and adds noise that helps escape local minima.

for i in range(iters):
    idx = np.random.choice(len(X), batch_size)  # random 64 indices
    Xb, yb = X[idx], y[idx]
    yh = self.forward(Xb)
    if i % 200 == 0: losses.append(bce(yb, yh))  # record every 200
    self.backward(Xb, yb)

HOW Decision Boundary Plot

Create a fine grid over the input space → run predictions on every grid point → colour regions by class. Visually shows what the network learned.

xx, yy = np.meshgrid(np.linspace(xmin, xmax, 200),
                     np.linspace(ymin, ymax, 200))
grid = np.c_[xx.ravel(), yy.ravel()]  # shape (40000, 2)
Z = model.predict(grid).reshape(xx.shape)
ax.contourf(xx, yy, Z, alpha=0.3)    # coloured regions
ax.scatter(X[:,0], X[:,1], c=y, s=10) # actual data points
P5 · Week 5
Multi-class NN — Variable-Depth, Softmax Output
"Go from 2 classes to 4. From hardcoded layers to a list-driven architecture."

WHEN Use This Pattern

When you have more than 2 classes. The output layer needs softmax (not sigmoid) and the target must be one-hot encoded. The layer sizes are passed as a list so you can change architecture without rewriting the class.

HOW The Variable-Depth Design

Instead of hardcoding W1, W2, W3 etc., you pass layers=[6,64,32,16,4] and the __init__ creates len(layers)-1 weight matrices automatically.

for i in range(len(layers)-1):
    self.W.append(np.random.randn(layers[i], layers[i+1]) * 0.1)
    self.b.append(np.zeros((1, layers[i+1])))
# layers=[6,64,32,16,4] creates:
# W[0]: (6,64), W[1]: (64,32), W[2]: (32,16), W[3]: (16,4)

HOW Forward Pass for Variable Layers

def forward(self, X):
    self.Z = []; self.A = [X]        # A[0] = input
    for i in range(len(self.W)-1):   # all layers except last
        z = self.A[-1] @ self.W[i] + self.b[i]
        self.Z.append(z)
        self.A.append(relu(z))       # ReLU for hidden layers
    z = self.A[-1] @ self.W[-1] + self.b[-1]
    self.Z.append(z)
    self.A.append(softmax(z))        # softmax for output

HOW Softmax — What and Why

Converts a vector of raw scores into probabilities that sum to 1. Always subtract the max first to prevent numerical overflow.

def softmax(x):
    e = np.exp(x - np.max(x, axis=1, keepdims=True))  # stability trick
    return e / np.sum(e, axis=1, keepdims=True)
# Output: [0.1, 0.7, 0.15, 0.05] — probabilities per class

HOW Backward Pass (Reversed Loop)

The key insight: the last layer gets dZ = loss_grad directly (softmax+CCE gradient simplifies to yh - y). All hidden layers additionally multiply by the ReLU derivative.

for i in reversed(range(len(self.W))):
    dZ = dA if i==len(self.W)-1 else dA * relu_d(self.Z[i])
    self.W[i] -= self.lr * (self.A[i].T @ dZ)
    self.b[i] -= self.lr * np.sum(dZ, axis=0, keepdims=True)
    dA = dZ @ self.W[i].T   # pass gradient to previous layer

HOW MSE vs CCE for Classification

LossFormulaGradientBest for
MSEmean((y−yh)²)(yh−y)/mRegression; works for classification too but slower
CCE−mean(Σ y·log(yh))(yh−y)/mMulti-class classification — much faster convergence

Both have the same gradient formula here! The difference is in the loss value itself (CCE punishes wrong confident predictions harder) and therefore how the training evolves.

HOW SGD vs Mini-batch (batch_size parameter)

# batch_size=1 → SGD: one sample at a time, very noisy
# batch_size=40 → Mini-batch: 40 samples, smoother gradient
for i in range(0, len(X), batch_size):
    xb, yb = X[i:i+batch_size], y[i:i+batch_size]

The only difference between SGD and mini-batch in this code is the batch_size value. No other code changes.

P6 · Week 6
5 Optimizers — SGD, Momentum, RMSprop, Adam, Adagrad
"All optimizers agree on the gradient. They only disagree on how to use it."

WHEN Use Each Optimizer

OptimizerUse WhenWeakness
SGDBaseline; simple problemsSlow, gets stuck in saddle points
MomentumSGD is too slow or oscillatingCan overshoot
AdagradSparse features (NLP)Learning rate shrinks to 0 over time
RMSpropRNNs, non-stationary problemsNo bias correction
AdamDefault choice for most problemsCan overfit; needs lower lr

HOW The State Pattern

Each optimizer needs memory (velocity, running averages). You create a state dictionary with zeros before training, then pass it to the step function each iteration so it can update and remember.

# Before training loop:
state = init_state(opt, params)   # e.g. {"v": {k:0}} for Momentum

# Inside loop:
params, state = step(opt, params, grads, state, lr)

HOW SGD — Simplest

# No state needed. Just subtract scaled gradient.
w -= lr * grad

HOW Momentum — Add Inertia

Keeps a velocity vector. Accumulates gradients over time so it keeps moving in a consistent direction (like a ball rolling downhill gaining speed).

# State: v = 0 initially
v = beta * v - lr * grad   # beta=0.9: keep 90% of old velocity
w += v
# If grad keeps pointing same direction → v grows → faster descent
# If grad reverses → v slows → dampens oscillations

HOW Adagrad — Per-Parameter Learning Rates

Accumulates the sum of squared gradients. Parameters that received large gradients get a smaller effective lr. Good for sparse data where some features rarely update.

# State: s = 0 initially (accumulated squared gradients)
s += grad**2                       # s only ever grows
w -= lr * grad / (sqrt(s) + eps)   # big s → small step
# Problem: s never shrinks, so lr → 0 eventually

HOW RMSprop — Fix Adagrad's Problem

Instead of accumulating all past gradients, uses an exponential moving average. Old gradients decay, recent ones matter more.

# State: s = 0 initially
s = beta*s + (1-beta)*grad**2      # beta=0.9: weighted avg
w -= lr * grad / (sqrt(s) + eps)   # s stays relevant, doesn't collapse

HOW Adam — Best of Both Worlds

Combines Momentum (1st moment = running mean of gradient) and RMSprop (2nd moment = running mean of squared gradient). Adds bias correction because m and v start at 0 (biased toward zero at start).

# State: m=0, v=0, t=1
m = b1*m + (1-b1)*grad           # 1st moment (like momentum)
v = b2*v + (1-b2)*grad**2       # 2nd moment (like RMSprop)

m_hat = m / (1 - b1**t)          # bias correction (t=step count)
v_hat = v / (1 - b2**t)

w -= lr * m_hat / (sqrt(v_hat) + eps)
t += 1

# b1=0.9, b2=0.999, eps=1e-8 are standard hyperparameters
Why bias correction? At step 1, m = 0.9×0 + 0.1×grad = 0.1×grad. This underestimates the true mean. Dividing by (1−0.9¹) = 0.1 corrects it back to grad. As t increases, (1−b1^t) → 1 so correction fades.
P8 · Week 8
Overfitting + Regularization (L1, L2, Dropout)
"A model that memorises the training data is useless. Regularization teaches it to generalise."

WHEN You See Overfitting

Train accuracy → 99%, validation accuracy → 60%. Training loss keeps falling, validation loss starts rising. This is overfitting — the model is memorising noise, not learning patterns.

In P8, overfitting is deliberately forced by using a very small training set (test_size=0.8 means only 20% is used for training).

HOW Dropout (Inverted Dropout)

Randomly zero out neurons during training. Forces the network to not rely on any one path. At test time, dropout is OFF (rate=0), so all neurons contribute.

# Training forward pass
if dropout > 0:
    mask = (np.random.rand(*A1.shape) > dropout) / (1-dropout)
    # random.rand > 0.3 → ~70% of neurons survive (rate=0.3)
    # divide by (1-rate) → "inverted dropout" scales up survivors
    #   so the expected value of A1 stays the same at test time
    A1 *= mask    # zero out the dropped neurons

# Testing forward pass (dropout=0, no mask applied)
_, _, _, y_prob, _ = forward(X_test, W1, b1, W2, b2, 0)

In backward, you must apply the same mask to dA1 — only the neurons that were active in forward pass should receive gradients.

HOW L2 Regularization (Weight Decay)

Adds a penalty to the loss equal to the sum of squared weights. This discourages large weights, forcing the model to spread weight evenly rather than rely on a few features.

# In loss:
l2_term = l2 * (np.sum(W1**2) + np.sum(W2**2)) / m
loss = bce + l2_term

# In backward (extra term added to dW):
dW2 += (l2 * W2) / m   # gradient of W² is 2W, but 2 is absorbed in l2 value
# Effect: weights are pulled toward 0 every step (weight decay)

HOW L1 Regularization (Sparsity)

Adds penalty equal to sum of absolute values of weights. Encourages weights to become exactly 0 (sparse). Good when you suspect many features are irrelevant.

# In loss:
l1_term = l1 * (np.sum(np.abs(W1)) + np.sum(np.abs(W2))) / m

# In backward:
dW2 += (l1 * np.sign(W2)) / m   # gradient of |W| is sign(W)
# sign(W) = +1 if W>0, -1 if W<0 → always pushes W toward 0

WHY L1 vs L2 vs Dropout

MethodEffect on WeightsWhen to Use
L2Shrinks all weights smoothly toward 0; none become exactly 0Default choice; works almost everywhere
L1Forces many weights to exactly 0 (sparse model)Feature selection; when you want to ignore irrelevant inputs
DropoutDoesn't change weight magnitudes; disrupts co-adaptation of neuronsDeep networks; especially with large hidden layers
Analogy: L2 = give everyone on the team a small pay cut. L1 = fire the least useful people entirely. Dropout = randomly remove people each day so nobody becomes irreplaceable.

HOW The train_model(l1, l2, dropout) Pattern

The entire training loop is wrapped in a function that accepts regularization hyperparameters. This makes it trivial to run all 8 variants by just calling it with different arguments.

variants = {
    "No_Reg":  {"l1":0,     "l2":0,     "dropout":0  },
    "L1":      {"l1":0.001, "l2":0,     "dropout":0  },
    "L2":      {"l1":0,     "l2":0.001, "dropout":0  },
    "Dropout": {"l1":0,     "l2":0,     "dropout":0.3},
}
for name, p in variants.items():
    W1,b1,W2,b2,tl,vl,ta,va = train_model(**p)
Core Concept
Backpropagation — The Full Mental Model
"If you understand this once, every P4–P8 backward pass writes itself."

THE BIG PICTURE

Forward pass computes a prediction and a loss. Backward pass asks: "how much did each weight contribute to that loss?" That answer is the gradient, computed via the chain rule — working backwards through every operation.

FORWARD: X → Z1 → A1 → Z2 → A2 → Loss BACKWARD: dLoss → dZ2 → dW2,db2 → dA1 → dZ1 → dW1,db1

HOW Each Step and Why

## FORWARD ##
Z1 = X @ W1 + b1       # linear: each neuron computes weighted sum
A1 = activate(Z1)      # non-linear: squash through activation
Z2 = A1 @ W2 + b2      # output layer linear
A2 = sigmoid(Z2)       # output probability

## BACKWARD (binary classification, BCE loss) ##
dZ2 = A2 - y           # gradient of BCE+sigmoid simplifies to this!
                       # (this is the beautiful simplification)

dW2 = A1.T @ dZ2 / m  # how much W2 contributed to loss
db2 = sum(dZ2) / m    # how much b2 contributed

dA1 = dZ2 @ W2.T      # backprop error signal to hidden layer
dZ1 = dA1 * act'(Z1)  # chain rule: multiply by activation derivative
                       # act'(Z1) "gates" how much gradient passes through
dW1 = X.T @ dZ1 / m  
db1 = sum(dZ1) / m    

## UPDATE ##
W -= lr * dW; b -= lr * db

WHY dZ2 = A2 - y (not a long formula)

Normally you'd have: dLoss/dZ2 = dLoss/dA2 × dA2/dZ2. For BCE loss with sigmoid output, these two derivatives cancel beautifully, leaving just A2 - y. This is why sigmoid+BCE and softmax+CCE are standard pairs — their gradients simplify cleanly.

TRICK How to Read Matrix Dimensions in Backprop

VariableShapeWhy
X(m, n_in)m samples, n_in features
W1(n_in, n_hid)connects each input to each hidden
Z1, A1(m, n_hid)one activation per sample per neuron
dW1 = X.T @ dZ1 / m(n_in, n_hid)must match W1 shape
dA1 = dZ2 @ W2.T(m, n_hid)must match A1 shape

Golden rule: every gradient must be the same shape as the variable it belongs to. If dW1 should be (n_in, n_hid), and dZ1 is (m, n_hid), then you need X.T which is (n_in, m) → gives (n_in, n_hid). ✓

OTS SURVIVAL TIPS

  • Start with the template: Write __init__, forward, backward, train, predict — in that order always.
  • Always seed: np.random.seed(42) before weight init for reproducibility.
  • Shape check habit: If code crashes, print X.shape, W1.shape, Z1.shape — dimensions will reveal the error.
  • Sigmoid output always needs reshape: y = y.reshape(-1,1) before training.
  • Multiclass needs argmax: np.argmax(model.forward(X), axis=1) to get predicted class.
  • Regularization = 2 additions only: Add penalty to loss function, add sign/W term to dW in backward. That's it.
  • Dropout = 1 addition only: Create mask in forward, apply same mask to dA1 in backward.
